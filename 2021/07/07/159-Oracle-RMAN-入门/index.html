¶ RMAN 环境概述

恢复管理器 (RMAN) 是一个 Oracle 数据库客户端，可对您的数据库执行备份和恢复任务并自动管理您的备份策略。它极大地简化了备份、还原和恢复数据库文件。

RMAN 环境由用于备份数据的实用程序和数据库组成。 RMAN 的环境至少必须包括以下组件：

  • 一个目标数据库: RMAN 使用 TARGET 关键字连接到的 Oracle 数据库。目标数据库是 RMAN 在其上执行备份和恢复操作的数据库。RMAN 始终在数据库的控制文件中维护有关其对数据库的操作的元数据。RMAN 元数据被称为 RMAN 存储库。
  • RMAN 客户端: 一个 Oracle 数据库可执行文件，用于解释命令、引导服务器会话执行这些命令，并在目标数据库控制文件中记录其活动。 RMAN 可执行文件随数据库自动安装，通常与其他数据库可执行文件位于同一目录中。例如，Linux 上的 RMAN 客户端位于 $ORACLE_HOME/bin 中。

某些环境使用以下可选组件：

  • 快速恢复区: 数据库可以在其中存储和管理与备份和恢复相关的文件的磁盘位置。您可以使用 DB_RECOVERY_FILE_DEST 和 DB_RECOVERY_FILE_DEST_SIZE 初始化参数设置快速恢复区的位置和大小。
  • 介质管理器: RMAN 与顺序介质设备（如磁带库）交互所需的应用程序。介质管理器在备份和恢复期间控制这些设备，管理介质的加载、标记和卸载。介质管理设备有时称为 SBT（系统备份到磁带）设备。
  • 恢复目录: 一种单独的数据库模式，用于记录针对一个或多个目标数据库的 RMAN 活动。如果控制文件丢失，恢复目录会保留 RMAN 存储库元数据，从而在控制文件丢失后更容易恢复和恢复。数据库可能会覆盖控制文件中的旧记录，但 RMAN 会在目录中永远保留记录，除非用户删除这些记录。

本章介绍了如何在没有恢复目录或介质管理器的最基本配置中使用 RMAN。
